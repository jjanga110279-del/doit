HTML의 기본 문법

HTML문서는 웹페이지의 뼈대 역할을 함
계층구조를 가진다

태그의 사용법

태그는 여는태그와 닫는태그로 구성됨

태그를 표시하는 기호는 < > 이며 괄호내에 명령어를 사용한다. 공백을 표시할때는            를 사용한다.

태그의 분류

태그가 웹페이지에서 차지하는 영역에 따라 두가지로 나눌 수 있다.

Block Level vs Inline Level 비교

항목 Block Level (블록 레벨) Inline Level (인라인 레벨)
너비 (Width) 웹페이지의 화면 전체 너비를 차지 콘텐츠의 크기만큼만 차지
줄바꿈 항상 새로운 줄에서 시작하고 끝남 텍스트 흐름 안에서 같은 줄에 배치
마진 (Margin) 위, 아래, 좌, 우 모두 적용됨 좌, 우만 적용 (위, 아래는 무시)
패딩 (Padding) 모든 방향 적용됨 좌, 우만 적용 (위, 아래 제한적)
높이 (Height) width, height 속성 적용 가능 width, height 속성 무시됨
정렬 text-align, margin으로 정렬 가능 text-align으로만 정렬 가능

주요 태그 분류

Block Level 태그 Inline Level 태그
태그 설명 태그 설명
<div> 일반 컨테이너 <span> 일반 인라인 컨테이너
<p> 문단 <a> 하이퍼링크
<h1~h6> 제목 <strong> 강한 강조 (굵음)
<ul>, <ol> 리스트 <em> 강조 (기울임)
<li> 리스트 항목 <img> 이미지
<table> 표 <button> 버튼
<form> 폼 <input> 입력 필드
<header> 헤더 영역 <label> 폼 레이블
<footer> 푸터 영역 <br> 줄바꿈
<section> 섹션 <i>, <b> 기울임, 굵음

인라인 vs 블록 사용 시 주의할 점

1️⃣ 블록 태그 내부에 인라인 태그 배치 (OK)

✓ 올바른 사용

블록 태그 안에 인라인 태그를 넣을 수 있습니다.

<div>
  <span>텍스트</span>
  <strong>강조</strong>
</div>

2️⃣ 인라인 태그 내부에 블록 태그 배치 (주의!)

✗ 잘못된 사용

인라인 태그 안에 블록 태그를 넣으면 레이아웃이 깨질 수 있습니다.

<span>
  <div>텍스트</div>  <!-- 문제 발생! -->
</span>

문제점: 브라우저가 자동으로 태그를 수정하거나 예상치 못한 결과가 발생합니다.

3️⃣ 인라인 태그에 높이/너비 설정 (주의!)

✗ 주의 필요

인라인 태그에 width, height를 설정해도 적용되지 않습니다.

<span style="width: 200px; height: 100px;">
  텍스트 (너비/높이가 적용 안됨!)
</span>

해결 방법: CSS의 display: block; 또는 display: inline-block;을 사용합니다.

✓ 올바른 해결

<span style="display: inline-block; width: 200px; height: 100px;">
  텍스트 (이제 너비/높이가 적용됨!)
</span>

4️⃣ 인라인 태그에 마진(위/아래) 설정 (주의!)

✗ 주의 필요

인라인 태그에 위/아래 마진을 설정해도 적용되지 않습니다.

<span style="margin-top: 20px; margin-bottom: 20px;">
  텍스트 (위/아래 여백이 적용 안됨!)
</span>

좌/우 마진은 정상 적용됨

✓ 올바른 해결

<span style="display: inline-block; margin-top: 20px; margin-bottom: 20px;">
  텍스트 (이제 모든 마진이 적용됨!)
</span>

5️⃣ 블록 태그 사용 시 불필요한 공백 주의

✗ 잘못된 사용

블록 태그들 사이의 공백이 마진과 겹칠 수 있습니다. (마진 병합 현상)

<div style="margin-bottom: 20px;">첫 번째</div>
<div style="margin-top: 20px;">두 번째</div>
<!-- 실제 간격은 40px이 아닌 20px! -->

6️⃣ 시맨틱 태그 사용 권장

✓ 권장 사용

의미있는 블록 태그를 사용하면 가독성과 SEO가 향상됩니다.

<!-- 좋음 -->
<header></header>
<nav></nav>
<section></section>
<article></article>
<footer></footer>

<!-- 피하기 -->
<div></div>
<div></div>
<div></div>

7️⃣ 인라인 태그로 레이아웃 구성하지 않기

✗ 주의 필요

페이지의 구조적 레이아웃은 반드시 블록 태그로 작성해야 합니다.

<!-- 절대 하면 안됨! -->
<span><span><span>레이아웃</span></span></span>

✓ 올바른 방법

<!-- 올바른 구조 -->
<header></header>
<main></main>
<aside></aside>
<footer></footer>

정리 및 요약

주의 사항 문제점 해결 방법
인라인 태그의 높이/너비 width, height 무시됨 display: inline-block; 사용
인라인 태그의 위/아래 마진 margin-top/bottom 무시됨 display: inline-block; 사용
인라인 안에 블록 태그 레이아웃 깨짐 블록 태그를 바깥으로 이동
마진 병합 현상 예상과 다른 간격 padding 사용 또는 마진 통일
인라인으로 레이아웃 유지보수 어려움 블록 태그 사용 (div, section 등)
의미 없는 div 남용 SEO 저하, 가독성 감소 시맨틱 태그 사용 (header, nav 등)
